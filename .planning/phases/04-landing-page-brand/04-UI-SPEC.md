### Copy Framework (Harry Dry — for Headline, Subheadline, CTA)

Apply these three tests to all discretionary copy:

1. **Can I visualize it?** Close your eyes and see it. "Optimize your profile" ✗ / "14,200 React developers on Malt — here's where you're rare" ✓
2. **Can I falsify it?** Point, don't talk. Show a number, name, situation. "Great for freelancers" ✗ / "366 Malt users claim this skill" ✓
3. **Can nobody else say it?** Only this tool surfaces Malt occurrence counts. Copy must own that uniqueness. "Search keywords" ✗ / "See what skills are worth thousands of profiles on Malt" ✓

### Discretionary Copy (Claude to write, applying Harry Dry framework)

| Element                  | Requirement                                                                                                                           | Example Approach                                                                                                   |
| ------------------------ | ------------------------------------------------------------------------------------------------------------------------------------- | ------------------------------------------------------------------------------------------------------------------ |
| Hero Headline            | Must reference Malt freelancers + pain point (wrong keywords = invisible = zero contacts). Concrete, falsifiable, only-we-can-say-it. | "See which keywords are actually searched on Malt" or "3,000+ Malt freelancers use this to find their rare skills" |
| Hero Subheadline         | Reinforce the concrete value without fluff. 1–2 sentences max.                                                                        | "Type any skill and instantly see how many Malt users claim it. Find the gaps where you stand out."                |
| CTA Button Label         | Must name the action + outcome. Avoid generic "Sign Up" / "Get Started".                                                              | "Find My Keywords" or "Search My Niche" or "See My Competition"                                                    |
| Primary CTA (below hero) | Scroll nudge if clicked before email gate fires. Encourages first search.                                                             | "Try it — search any skill" (shown as toast or inline after click)                                                 |

**Tone Across All Copy:**

- Blunt and short. "One Mississippi test" — if taking >2 seconds to read, rewrite.
- Concrete language only. Numbers, skills, situations.
- No jargon ("leverage," "optimize," "synergy").

- Layout: Flex column, gap 12px (sm) or 16px (md)

  - Background: white
  - Border: 1px solid #D1D5DB (gray-300)
  - Border radius: 8px (rounded)
  - Padding: 12px (sm) or 16px (md)
  - Hover: `bg-gray-50`, shadow-sm
  - Animation: `fade-in` with stagger (see Animations section)

**Rank Badge (on left side of each card):**

- Style: Circular, 24px diameter
- Background: #6366F1 (Indigo)
- Text: "#1", "#2", "#3", "#4"
- Color: white
- Font: 12px, weight 700 (bold)
- Centered text
- Margin right: 12px (sm)

**Card Content (flex row, items-center):**

- Left: Rank badge (circular, 24px)
- Middle: Keyword label (16px, weight 500, gray-900)
- Right: Volume count (14px, weight 400, gray-600) — format: "{count} utilisateurs Malt"
- Competition badge (existing pattern from KeywordCard): small badge showing rare/common/oversaturated

**Cards Fetch Strategy:**

- 4 parallel API calls to `/api/malt/autocomplete` on page mount
- Seed keywords: 1 random per niche (tech, design, project, devops)
- Each seed selected randomly from predefined pool per niche (5–6 options per niche)
- Load state: Skeleton loaders (4 cards, same layout as real cards)
- Error state: Fallback to hardcoded popular keywords (no crash)

## Animations & Transitions

### Leaderboard Card Stagger (on page load)

Use CSS @keyframes in globals.css. Each of 4 cards fades in with 100ms stagger:

```css
@keyframes fadeInUp {
  from {
    opacity: 0;
    transform: translateY(8px);
  }
  to {
    opacity: 1;
    transform: translateY(0);
  }
}

.leaderboard-card {
  animation: fadeInUp 300ms ease-out forwards;
}

.leaderboard-card:nth-child(1) {
  animation-delay: 0ms;
}
.leaderboard-card:nth-child(2) {
  animation-delay: 100ms;
}
.leaderboard-card:nth-child(3) {
  animation-delay: 200ms;
}
.leaderboard-card:nth-child(4) {
  animation-delay: 300ms;
}
```

Or use Tailwind's `animate-pulse` during skeleton load state.

### Post-Verification Success Fade

Fade-in (300ms ease-out), hold for 2–3 seconds, then fade-out (300ms ease-out):

```css
@keyframes fadeIn {
  from {
    opacity: 0;
  }
  to {
    opacity: 1;
  }
}

@keyframes fadeOut {
  from {
    opacity: 1;
  }
  to {
    opacity: 0;
  }
}

.success-state {
  animation:
    fadeIn 300ms ease-out,
    fadeOut 300ms ease-out 2500ms forwards;
}
```

### Button Hover & Focus

Existing patterns from SearchInput:

- Hover: Darken background, subtle shadow
- Focus: Ring 2px with offset (already applied in SearchInput)
